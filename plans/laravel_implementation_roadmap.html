# Implementation Guide: Laravel Portfolio Architecture Building your personal portfolio with Laravel is a great way to showcase your expertise. Since we've already established the "Neon Logic" design system and 5 core screens, here is the roadmap to turn them into a dynamic application. ## 1. Technical Stack Recommendation * **Framework:** Laravel 11.x * **Frontend:** Inertia.js with Vue.js (matches the high-interactivity of your designs) or Blade + Alpine.js (for a more traditional PHP-centric approach). * **Styling:** Tailwind CSS (all generated HTML uses Tailwind classes). * **Database:** MySQL or PostgreSQL (as seen in your "Technical DNA" section). ## 2. Core Data Models To make the site dynamic, you'll want to move content out of static HTML and into the database: * **Project:** `title`, `description`, `slug`, `image_path`, `github_url`, `live_url`, `tags` (JSON or many-to-many), `category` (SaaS, API, etc.). * **Post:** `title`, `body` (Markdown support recommended), `slug`, `excerpt`, `published_at`, `category_id`. * **Skill/Technology:** `name`, `icon_class`, `category` (Frontend, Backend, DevOps). * **Inquiry:** `name`, `email`, `message`, `ip_address` (for the Contact form). ## 3. Implementation Steps ### Phase 1: Environment Setup 1. Initialize a new Laravel project: `laravel new portfolio`. 2. Install Tailwind CSS and configure it with the custom colors from your design system (`#BD00FF`, etc.). 3. Import the **Space Grotesk** and **Inter** fonts. ### Phase 2: The Layout & Components 1. **Shared Layout:** Create a base `app.blade.php` or `AuthenticatedLayout.vue` using the `TopAppBar` and `Footer` components defined in the DataStore ({{DATA:COMPONENTS:COMPONENTS_3}}). 2. **Blade Components:** Break down repeating elements like the "Technical DNA" cards and "Selected Artifacts" cards into reusable Laravel components. ### Phase 3: Dynamic Routing & Controllers 1. **HomeController:** Serves the hero section and a "latest 3" preview of projects/posts. 2. **ProjectController:** Lists all projects and shows details. 3. **BlogController:** Handles the article listing and the specific deep-dive view ({{DATA:SCREEN:SCREEN_4}}). 4. **ContactController:** A simple `POST` route to store messages and perhaps trigger a Filament or Mail notification to you. ### Phase 4: Content Management (Optional but Recommended) Since you're a senior dev, you might not need a full CMS, but adding **Laravel Filament** would give you a sleek admin panel to manage your portfolio and blog posts without touching the database directly. ## 4. Design-to-Code Tips * **SVG Assets:** Export the neon icons and grid patterns from the HTML code to your `/public/assets` folder. * **Animations:** Use **Framer Motion** (if using Vue) or **Alpine.js** to add the subtle "flicker" and "fade-in" effects typical of the synthwave aesthetic. * **Code Highlighting:** Use a library like `Spatie/laravel-markdown` combined with `Shiki` to get the beautiful syntax highlighting seen in the Blog layout. --- **Next Step:** Would you like me to generate a specific set of Laravel Blade templates or a sample Migration file for these models to get you started?
